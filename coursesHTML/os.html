Intro. to Operating System

As we know in computer technology there is two types of sowftware exists


  1. Application Software : An Application software is designed to perform a specific task and these programs installs in Operating System. (will we discuss some more talks on Application Software later in this course)

  2. System Software : System Softwares are some special type of software that directly deals with hardware. e.g. System softwares are designed to operate computer hardware, peripherial devices etc, System softwares are that softwares which provides us the feature of plug and play.


Note: Operating System is an special type of system software that directly deals with hardware


What is an Operating System ?

Operating System : An Operating System is a piece of software that manages all the resources of a computer system both hardware and software and provides an environment in which users can perform tasks by executing the applications or processes in a convenient and efficient manner. OS hides the complexity of the hardware by working as resource manager.

Operating System Installed directly on hardware and makes a efficient environment to use the functionality of hardware, then a end user can use it without any complexity

Layered structure of operating system

  • Layer 1: Computer Hardware.

  • Layer 2: CPU Scheduling.

  • Layer 3: Memory Management.

  • Layer 4: Process Management

  • Layer 5: Operating System.

  • Layer 6: User programs / Application Software3

Objectives Of Operating System

The main targets of an operating system are as follows :


  • Convenience :An operating system improves the use of a Hardware. Operating systems gives the ability to users to use systems without the stress of configuring system and resource management.

  • Efficiency :An Operating System has the ability to manage all the resources properly and use them as needed. This is due to less time spent on configuring the system.

  • Ability to evolve :An operating system should be designed in such a way that it allows for the effective development, testing, and introduction of new features without interfering with service.

  • Management Of System Resources :Operating systems are able to share resources fairly among various processes that are running and user.

Functions Of Operating System

  1. Memory Management:

    Operating System Manages the primary or main memory. If any program is executed, it has to be present in the memory. Main memory is faster than Secondary memory and directly accessed by the CPU. When a program is executed the operating system allocates the required memory to the process of it and by the CPU it handles the process in an efficient manner after the completion of the process Operating System deallocates the memory and free up the space in main memory for another process.

    • Allocates and Deallocates the memory.

    • Keeping the record of which part of memory is used and which process is using.

    • Distributing the memory while multiprocessing etc.

  2. CPU Management or Scheduling:

    Every software that runs on a computer, whether in the background or in the frontend, is a process. Processor management is an execution unit in which a program operates. The operating system determines the status of the processor and processes, selects a job and its processor, allocates the processor to the process, and de-allocates the processor after the process is completed

    When more than one process runs on the system the OS decides how and when a process will use the CPU. Hence, the name is also CPU Scheduling The OS:

    • Manages the CPU uses by process.

    • Keeps the record of CPU status

    Certain algorithms used for CPU Scheduling in OS are as follows :

    • First Come First Out.

    • Shortest Job First.

    • Round-Robin Scheduling.

    • Priority-based scheduling etc

    Purpose of CPU scheduling:

    • Proper utilization of CUP is necessary. Operating System makes sure that the CPU should be as busy as possible

    • Since every device should get a chance to use the processor. Hence, the OS makes sure that the devices get fair processor time

    • Increasing the efficiency of the system.

    • Allocates and deallocates devices to different processes

    • Keeps records of the devices

    • Decides which process can use which device for how much time

  3. File Management:

    The operating system manages resource allocation and deallocation. It specifies which process receives the file and for how long. It also keeps track of information, location,uses,status, and so on. These groupings of resources are referred to as file systems. The files on a system are stored in different directories.

    The Operating System :

    • Keeps records of the status of and locations of files

    • Allocates and deallocates.

    • Decides who gets the resources.

  4. Storage management:

    Storage managementis defined as it refers to the management of the data storage equipment’s that are used to store the user/computer generated data. Hence it is a tool or set of processes used by an administrator to keep your data and storage equipment’s safe. Storage management is a process for users to optimize the use of storage devices and to protect the integrity of data for any media on which it resides and the category of storage management generally contain the different type of subcategories covering aspects such as security, virtualization and more, as well as different types of provisioning or automation, which is generally made up the entire storage management software market.

    The OS uses storage management for:

    • Improving the performance of the data storage resources.

    • It Optimizes the use of various storage devices.

    • Assists businesses in storing more data on exiting hardware, speeding up the data retrieval process, preventing, preventing data loss, meeting data retention regulations, and lowering IT costs.
